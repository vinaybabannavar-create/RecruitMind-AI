## 🚀 Overview

**RecruitMind AI** is a fully agentic, production-ready AI recruitment intelligence platform that fundamentally transforms how talent is discovered, evaluated, and shortlisted. Built on a **LangGraph stateful pipeline**, it moves far beyond keyword filtering — using semantic vector search, LLM-powered cognitive JD parsing, multi-signal composite scoring, and explainable AI reasoning to surface the *exact right candidates* for any job description.

Traditional ATS systems filter by keywords. **RecruitMind AI understands intent.**

```
You paste a Job Description → RecruitMind AI thinks like a senior recruiter → You get a ranked, explained shortlist in seconds.
```

### ✨ Key Highlights

| Feature | Description |
|---------|-------------|
| 🔍 **Cognitive JD Parsing** | LLM extracts skills, seniority, culture signals, and *implicit* requirements from raw JD text |
| 📊 **Multi-Signal Scoring** | Composite scores across 5 independent signals — not just keyword overlap |
| 💡 **Explainable AI** | Natural language recruiter notes per candidate, generated by Groq Llama-3.3-70B |
| 🎯 **Semantic Vector Search** | ChromaDB + Sentence Transformers (all-MiniLM-L6-v2) for deep semantic matching |
| ⚡ **LangGraph Pipeline** | Modular, observable, stateful agentic pipeline with 6 nodes |
| 🔁 **Diversity Re-ranking** | Prevents shortlist homogeneity — balances seniority and role type |
| 🌐 **Offline Fallback** | Works fully without an LLM key using rule-based parsing |
| 📥 **Export Ready** | Download shortlist as JSON or CSV |
| 🐳 **Docker Ready** | One command deployment with docker-compose |

## 🧠 LangGraph Agentic Pipeline

The entire ranking process is orchestrated as a **LangGraph stateful graph** — each stage is an independent, testable node that operates on a shared typed state.

```
                    ┌─────────────┐
                    │  [JD Input] │
                    └──────┬──────┘
                           │
                           ▼
              ┌────────────────────────┐
              │   Node 1: parse_jd    │  ← LLM extracts: job title, required skills,
              │                        │    seniority, culture signals, implicit needs
              └────────────┬───────────┘
                           │
                           ▼
              ┌────────────────────────┐
              │   Node 2: embed_jd    │  ← Sentence Transformer converts JD to
              │                        │    dense vector for semantic comparison
              └────────────┬───────────┘
                           │
                           ▼
              ┌────────────────────────┐
              │  Node 3: retrieve     │  ← ChromaDB cosine similarity search
              │                        │    returns top-N semantically similar candidates
              └────────────┬───────────┘
                           │
                           ▼
              ┌────────────────────────┐
              │  Node 4: score        │  ← 5-signal composite scoring per candidate
              │                        │    (semantic + skill + career + activity + exp)
              └────────────┬───────────┘
                           │
                           ▼
              ┌────────────────────────┐
              │  Node 5: rerank       │  ← Diversity-aware reranking to prevent
              │                        │    shortlist homogeneity
              └────────────┬───────────┘
                           │
                           ▼
              ┌────────────────────────┐
              │  Node 6: explain      │  ← Groq Llama-3.3 generates recruiter-style
              │                        │    reasoning note for each candidate
              └────────────┬───────────┘
                           │
                           ▼
              ┌────────────────────────┐
              │  [Ranked Shortlist]   │  ← JSON: rank + score + breakdown + explanation
              └────────────────────────┘
```

**State is fully typed** using `TypedDict` — every node receives and returns the complete pipeline state, making it easy to inspect, debug, and extend any stage independently.

## 🔬 Multi-Signal Scoring Engine

RecruitMind AI does not rank candidates by a single metric. Every candidate receives a **composite score built from 5 independent signals**, each capturing a different dimension of fit.

| Signal | Weight | How It's Computed |
|--------|--------|--------------------|
| 🧬 **Semantic Similarity** | **35%** | Cosine distance between JD embedding and candidate profile embedding via ChromaDB |
| 🛠️ **Skill Match** | **30%** | Required skills hit rate (80%) + nice-to-have bonus (20%) |
| 📈 **Career Growth** | **15%** | Raw growth score × tenure stability × promotions bonus |
| ⚡ **Activity Signal** | **10%** | GitHub contributions + open source PRs + LeetCode + hackathons × recency multiplier |
| 📅 **Experience Fit** | **10%** | Linear fit to JD experience range + seniority level alignment multiplier |

```
Composite Score = (0.35 × Semantic) + (0.30 × Skill) + (0.15 × Career)
               + (0.10 × Activity) + (0.10 × ExperienceFit)
```

Scores are then passed through the **diversity re-ranker**, which applies a mild penalty to over-represented seniority levels or role types in the shortlist, ensuring variety.

## ⚙️ Setup & Installation

### Prerequisites

- Python 3.11+
- pip
- Git
- Groq API key — free at [console.groq.com](https://console.groq.com) *(optional — system works without it)*

---

### Step 1 — Clone the repository

```bash
git clone https://github.com/vinaybabannavar-create/-RecruitMind-AI.git
cd -RecruitMind-AI
```

### Step 2 — Create virtual environment

```bash
python -m venv venv

# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate
```

### Step 3 — Install dependencies

```bash
pip install -r requirements.txt
```

> **Note:** `sentence-transformers` will download the `all-MiniLM-L6-v2` model (~80MB) on first run. This is automatic.

### Step 4 — Configure environment

```bash
cp .env.example .env
```

Edit `.env`:

```env
# Get free key at https://console.groq.com
# Leave blank to run in rule-based offline mode
GROQ_API_KEY=your_groq_api_key_here

# Streamlit connects to this URL
API_URL=http://localhost:8000
```

### Step 5 — Generate candidate data

```bash
python data/generate_candidates.py
# Output: data/candidates.json (100 synthetic profiles)
```

### Step 6 — Index candidates into ChromaDB

```bash
python -c "from pipeline.embedder import get_engine; get_engine().index_candidates()"
# This embeds all 100 candidates and stores them in data/chroma_db/
```

**Request Body:**
```json
{

  "top_k": 10,
  "with_explanations": true
}
```

**Response:**

## 💡 What Makes RecruitMind AI Different

| Approach | Traditional ATS | RecruitMind AI |
|----------|----------------|----------------|
| JD Understanding | Keyword extraction | LLM-powered semantic parsing + implicit signal detection |
| Candidate Matching | `if skill in resume` | Cosine similarity over dense vector embeddings |
| Scoring | Binary filter (pass/fail) | 5-signal weighted composite score |
| Ranking | Alphabetical / date applied | Multi-dimensional intelligence score |
| Explanation | None | Per-candidate natural language recruiter note |
| Diversity | None | Built-in diversity re-ranker |
| Transparency | Black box | Full score breakdown per signal |
